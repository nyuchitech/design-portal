# Component Backlinks

Every component in the design system has canonical URLs on design.nyuchi.com. These serve as the component identity — documentation, playground, health check, and chaos test target.

## URL Structure

| Endpoint | URL Pattern | Purpose |
|---|---|---|
| Portal | design.nyuchi.com/components/\{name\} | Documentation page |
| API | design.nyuchi.com/api/components/\{name\} | JSON API |
| Playground | design.nyuchi.com/playground/\{name\} | Live playground |
| Health | design.nyuchi.com/api/health/\{name\} | Health check target |
| Source | design.nyuchi.com/source/\{name\} | Source code view |
| Changelog | design.nyuchi.com/changelog/\{name\} | Version history |
| Chaos | design.nyuchi.com/api/chaos/\{name\} | L8 chaos injection |
| Fundi | design.nyuchi.com/api/fundi/\{name\} | L9 healing trigger |

## DOM Integration

Every rendered component includes a data-portal attribute:

```html
<div data-slot="nyuchi-section" data-portal="https://design.nyuchi.com/components/nyuchi-section">
```

This allows:
- L8 chaos testing to target specific components by URL
- L9 Fundi to link healing actions to documentation
- DevTools inspection to click through to docs
- Error tracking to enrich errors with portal links

## Database

```sql
-- Get all links for a component
SELECT * FROM get_component_links('nyuchi-section');

-- Query all components with backlinks
SELECT * FROM component_backlinks;
```
