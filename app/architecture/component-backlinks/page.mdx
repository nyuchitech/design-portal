# Component Backlinks

Every component in the design system has a stable set of canonical URLs on design.nyuchi.com. The portal page, JSON API, source view, version history, per-component health probe, chaos manifest, and per-component Fundi surface are all live — every URL below resolves for any registry component name.

## URL Structure

| Endpoint | URL Pattern | Purpose |
|---|---|---|
| Portal | design.nyuchi.com/components/\{name\} | Documentation page |
| Source | design.nyuchi.com/source/\{name\} | Full TypeScript source view |
| Changelog | design.nyuchi.com/changelog/\{name\} | Per-component version history |
| API (component) | design.nyuchi.com/api/v1/ui/\{name\} | Component metadata + source (shadcn-format) |
| API (docs) | design.nyuchi.com/api/v1/ui/\{name\}/docs | Use cases, variants, a11y notes |
| API (versions) | design.nyuchi.com/api/v1/ui/\{name\}/versions | Per-component version JSON |
| Health | design.nyuchi.com/api/health/\{name\} | Per-component health probe (200/404/410/503) |
| Chaos | design.nyuchi.com/api/chaos/\{name\} | Chaos-fault manifest (GET; POST is 405 by design) |
| Fundi | design.nyuchi.com/api/fundi/\{name\} | Per-component Fundi issue list (GET) + report (POST) |

The site-wide health probe lives at `/api/v1/health`; the cross-cutting Fundi surface lives at `/api/v1/fundi`. Use the per-component URLs above when you have a target component name (and the URLs in any rendered `data-portal` attribute).

## Status semantics for /api/health/\{name\}

- `200` — component exists and is stable
- `404` — component does not exist
- `410` — component exists but is deprecated (reserved; the `components` table has no status enum yet)
- `503` — Supabase unconfigured (operator action required)

## Status semantics for /api/chaos/\{name\}

- `GET 200` — returns the supported-fault manifest for this component (render-error, slow-render, missing-data, network-timeout)
- `POST 405` — chaos injection is in-process by design; install the L8 chaos lib in your app rather than hitting the portal
- `404` / `503` — same as above

## DOM Integration

Every rendered component includes a `data-portal` attribute pointing at its live portal page:

```html
<div data-slot="nyuchi-section" data-portal="https://design.nyuchi.com/components/nyuchi-section">
```

This allows:
- DevTools inspection to click through to component docs
- Error tracking to enrich errors with portal links
- L8 chaos testing to look up the supported-fault manifest by stable URL
- L9 Fundi to link healing actions back to component documentation

## Database

```sql
-- Get all links for a component
SELECT * FROM get_component_links('nyuchi-section');

-- Query all components with backlinks
SELECT * FROM component_backlinks;
```
