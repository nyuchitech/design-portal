# Skills

Skills are reusable workflows AI assistants invoke on specific tasks. The Nyuchi Design Portal publishes three:

| Skill | When to invoke |
|---|---|
| `nyuchi-design-system` | Working on any bundu-ecosystem app (mukoko, nyuchi, shamwari, nhimbe) — the Five African Minerals palette, 3D frontend architecture, Ubuntu doctrine, CVA + Radix + cn pattern, APCA targets. |
| `scaffold-component` | Adding a new UI component to the registry: MCP `scaffold_component` → Supabase upsert → `pnpm registry:sync` → verify via `/api/v1/ui/{name}`. |
| `ecosystem-app-setup` | Bootstrapping a new Bundu/Mukoko ecosystem app from scratch — Next.js + shadcn + Tailwind 4 + next-themes against this registry. |

Skills are versioned MDX bodies stored in the Supabase `skills` table — that table is the single source of truth, served four ways for any consumer to load.

## Four install paths, one source

### 1. Nyuchi Design CLI

Always pulls live from the API; tracks installed versions in `.nyuchi-design.json`.

```bash
npx @nyuchi/design-cli skills install            # all skills
npx @nyuchi/design-cli skills install <name>     # one skill
npx @nyuchi/design-cli skills update             # diff against .nyuchi-design.json,
                                                  # rewrite changed skills only
```

### 2. Community skills CLI (npm bundle)

Offline-friendly. Installs the published snapshot from `@nyuchi/design-agent-skills`.

```bash
npx skills add @nyuchi/design-agent-skills
```

### 3. Claude Code plugin marketplace

If your editor is Claude Code:

```
/plugin marketplace add nyuchi/design-portal
/plugin install nyuchi-design
```

The plugin manifest at `.claude-plugin/plugin.json` declares the skills + the MCP server. Once installed, the skills auto-load on every project Claude Code opens.

### 4. Direct HTTP

```bash
curl https://design.nyuchi.com/api/v1/skills                      # list (no body_mdx)
curl https://design.nyuchi.com/api/v1/skills/summary               # light list, version drift
curl https://design.nyuchi.com/api/v1/skills/nyuchi-design-system  # full body_mdx
```

Returns the `SkillRow` shape: `{ name, description, body_mdx, agents, applies_to, requires_mcp, version, status, updated_at }`. CORS is permissive; cache 1h public / 1d s-maxage.

## MCP

For AI assistants connected to `https://design.nyuchi.com/mcp`:

| Tool | Description |
|---|---|
| `list_skills` | Returns every skill summary (no body_mdx). Use first to discover available skills. |
| `get_skill(name)` | Returns one skill including the full MDX body. |

Both wrap the same SQL helpers (`list_skills()`, `get_skill(name)`) the HTTP routes use.

## Compatibility

Each skill carries a YAML frontmatter `agents: [...]` array declaring which AI tooling it targets:

| Skill | Agents | Applies to |
|---|---|---|
| `nyuchi-design-system` | claude-code · cursor · copilot · cline · windsurf | next.js · astro · vite · react-native |
| `scaffold-component` | claude-code · cursor · copilot · cline | next.js · astro |
| `ecosystem-app-setup` | claude-code · cursor · copilot | next.js |

Skills with `requires_mcp: true` (the first two) work best when the consumer also has `https://design.nyuchi.com/mcp` configured — they reference live registry counts and component data via MCP tools.

## How drift is prevented

The four install paths all derive from `skills.body_mdx`:

```
Supabase `skills` table  ──┬──►  /api/v1/skills/{name}     (HTTP)
                            ├──►  MCP get_skill(name)        (Streamable HTTP)
                            ├──►  pnpm skills:sync           (regen .md files)
                            │       │
                            │       └──►  packages/design-agent-skills/skills/  ──►  npm publish
                            │
                            └──►  @nyuchi/design-cli skills install   (live fetch)
```

Before each `npm publish`, `pnpm skills:sync` regenerates the `.md` files from the live database. CI runs `pnpm skills:verify` (the `--check` mode) and fails the build if the on-disk files have drifted from the table — so a stale snapshot can never reach npm.

## Authoring a new skill

Skills live in Supabase, not in the file system. To add one:

```sql
INSERT INTO public.skills (name, description, body_mdx, agents, applies_to, requires_mcp, version, status)
VALUES (
  'your-skill-name',
  'When the AI should invoke this skill (one or two sentences).',
  '---
name: your-skill-name
version: 1.0.0
description: ...
agents: claude-code, cursor, copilot
requires_mcp: false
---

# Your Skill Name

The skill body in MDX.
',
  ARRAY['claude-code', 'cursor', 'copilot'],
  ARRAY['next.js'],
  false,
  '1.0.0',
  'published'
);
```

Then `pnpm skills:sync` and bump `@nyuchi/design-agent-skills` to publish the snapshot. The HTTP, MCP, and CLI paths pick up the new skill automatically — no code change required.
