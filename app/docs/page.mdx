# Nyuchi Design System

The Nyuchi Design System is the foundation of everything built by Nyuchi Africa. It powers Mukoko (Africa's super app), the Nyuchi Web Services developer platform, and every product in the ecosystem.

## Philosophy

Built on Ubuntu — "I am because we are." Every component, every token, every decision is made with the community in mind. The system is open source and designed for anyone to use, extend, and contribute to.

## The Five African Minerals

Our color palette draws from the continent's mineral wealth:

- **Cobalt** — Democratic Republic of Congo. Trust, depth, technology.
- **Tanzanite** — Tanzania. Rare, precious, transformation.
- **Malachite** — Zambia, DRC. Growth, success, nature.
- **Gold** — South Africa, Ghana. Value, warmth, achievement.
- **Terracotta** — Pan-African Sahel. Earth, craft, heritage.

For canonical hex values and container colours, query `brand_minerals` in the database — it's the source of truth and stays current as we refine the palette.

## Architecture

The system uses a 3D architecture model:

- **X-axis (Horizontal):** What users see — Primitives → Brand → Pages → Shell
- **Y-axis (Vertical):** What flows through — Tokens, Safety, Resilience
- **Z-axis (Depth):** What observes — Assurance (Layer 8: a11y audit, RTL conformity, probes, alerts)
- **Outside Actor:** Fundi (Layer 9) — the self-healing engine
- **Layer 10:** This documentation layer

See the 3D Architecture page for the full conceptual model.

## What's in the database

Everything that defines the design system lives in Supabase, not in scattered config files:

- **Components** — every UI component, with its production TypeScript source code in `components.source_code`. The frontend repo pulls from the DB at build.
- **Design tokens** — 18 brand_* tables covering minerals, semantic colours, spacing, typography, radius, shadow, motion, breakpoints, z-index, icon sizes, touch targets, density, component specs, and accessibility validation.
- **Documentation** — this page and others in `documentation_pages`.
- **AI instructions** — prompts for Claude, Cursor, Copilot, and the MCP server in `ai_instructions`.
- **Changelog** — every release in `changelog`.

To see the live system shape, run:

```sql
SELECT * FROM get_system_counts();
SELECT * FROM get_token_counts();
SELECT * FROM get_layer_counts();
```
